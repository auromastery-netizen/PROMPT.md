# CLAUDE.md — Auro AI Project Guide

## What This Repository Is

This is a **specification repository** for the Auro AI web system — an AI education platform for small business owners. The primary file, `PROMPT.md`, is a master build prompt that fully specifies two standalone HTML pages to be generated by AI coding tools (Claude, Cursor, Windsurf, etc.).

There is no compiled code, build system, or test suite. The deliverables are two vanilla HTML files produced by feeding `PROMPT.md` into an AI assistant.

---

## Project Goal

Build two self-contained single-file HTML pages:

| File | Purpose |
|------|---------|
| `index.html` | Marketing funnel — captures leads, redirects to dashboard |
| `dashboard.html` | Course player — 4 modules, 20 lessons, progress tracking |

**Constraint:** Both files must be fully self-contained — all CSS and JS inline inside `<style>` and `<script>` tags. No frameworks, no build tools. The only external dependency is Google Fonts.

---

## index.html — Implementation Notes

### Sections (top to bottom)
1. Sticky nav (68px) — logo left, trust badge right
2. Hero — two-column grid (copy left, video right); stacks on mobile ≤860px
3. Stats bar — 4 stats in Bebas Neue gold
4. "What You'll Learn" — 4 module preview cards
5. Instructor section — avatar + bio (placeholder initials "YN")
6. Lead capture form — validation + success state + redirect countdown
7. Bottom CTA — links directly to `dashboard.html`
8. Footer — links, copyright

### Form Behavior
- Validate: first name not empty, email matches regex
- On submit: show loading state ("Sending…")
- Default: demo mode — 1s timeout then `showSuccess()`
- `showSuccess()`: hide form, show countdown from 5→0, then `window.location.href = 'dashboard.html'`
- Meta Pixel `fbq('track', 'Lead')` call goes inside `showSuccess()` — **commented out by default**

### Backend Integration (all commented out, demo mode active)
```
Option A: Formspree
Option B: ConvertKit API
Option C: Custom POST to /api/leads
```

### Layout Structure
```
[Fixed header 62px — full width]
[Fixed sidebar 310px] | [Scrollable main content]
```
On mobile (≤768px): sidebar hidden, slides in on hamburger tap; main is full width.

### CSS Variables (set on `:root`)
```css
--bg: #07070f; --bg2: #0d0d1c; --bg3: #111124;
--gold: #F7C63A; --orange: #FF6B35;
--text: #e8e8f0; --muted: #6e6e8e;
--border: rgba(247,198,58,0.1); --border2: rgba(255,255,255,0.06);
--teal: #3ecfb2; --red: #ff6b6b; --purple: #a97cf8; --green: #4ade80;
--sidebar-w: 310px; --header-h: 62px;
```

### State Management
```javascript
let state = {
  started: false,
  currentModule: 0,   // 0–3
  currentLesson: 0,   // 0–4
  completed: {}       // keys: "0-1", "2-3" etc.
}
// Persisted to sessionStorage on every mutation
// Loaded from sessionStorage on page init
```

### Required JS Functions
```
buildSidebar()       — re-renders sidebar HTML from state
renderLesson()       — renders lesson view from state.currentModule + currentLesson
goToLesson(mi, li)   — updates state → renderLesson() + buildSidebar()
markComplete()       — marks done, routes to next lesson / module complete / course complete
beginCourse()        — started=true, goToLesson(0, 0)
beginNextModule()    — currentModule++, goToLesson(newModule, 0)
showModuleComplete() — celebration screen between modules
showCourseComplete() — final screen after lesson 3-4
toggleModule(mi)     — open/close sidebar module group
toggleSidebar()      — mobile sidebar show/hide
flatLessons()        — flat array of {mi, li, lesson} for prev/next nav
overallPct()         — 0–100 completion percentage
hexToRgb(hex)        — hex → "r,g,b" string for rgba() use
```

### Lesson View Structure
1. Breadcrumb: `Module Name › Lesson Title`
2. Video block (16:9) — placeholder with play button + HTML comments for YouTube/Vimeo/MP4
3. Module tag: `[emoji] [Module Name] · [Lesson Tag]` in module accent color
4. `<h1>` lesson title (Bebas Neue ~2.5rem)
5. Meta row: duration, lesson number, "Completed" badge
6. Lesson body HTML: h3 headings, paragraphs, bullet lists (✦ markers), callout boxes, key takeaway boxes
7. Bottom nav: ← Prev | Mark Complete & Continue → | Next →

### Fade Animation
```css
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(16px); }
  to   { opacity: 1; transform: none; }
}
.fade-in { animation: fadeIn 0.4s ease both; }
```
Apply `.fade-in` to lesson content wrapper on every render call.

## Development Workflow

Since there is no build system:

1. **Edit** `index.html` or `dashboard.html` directly
2. **Preview** by opening the file in a browser (`file://` or a local HTTP server)
3. **Test** the full funnel flow: landing page form → redirect → dashboard → complete lessons
4. **Commit** changes with descriptive messages on the feature branch
5. **Push** to `origin` when ready

No `npm install`, no compilation, no test runner. Open the HTML file and inspect it.

### Verifying the Dashboard
- First visit shows welcome screen; clicking "Begin Module 1 →" starts the course
- Completing a lesson advances state and re-renders sidebar + content
- Completing the last lesson in a module shows the Module Complete screen
- Completing all 20 lessons shows the Course Complete screen
- Refreshing the page should restore the last position (sessionStorage)
- On mobile, the hamburger should toggle the sidebar overlay

## Key Conventions for AI Assistants

- **Single-file rule:** Never split CSS or JS into separate files. Everything stays in `<style>` and `<script>` tags within each HTML file.
- **No frameworks:** Vanilla HTML/CSS/JS only. No React, Vue, Tailwind, jQuery, etc.
- **Brand consistency:** Always use the exact hex values from the design system. Do not substitute or approximate colors.
- **Font usage:** Bebas Neue for all display/heading text; DM Sans for all body/UI text. No exceptions.
- **State via sessionStorage:** The dashboard persists state in `sessionStorage` (not localStorage) so progress resets on new browser sessions.
- **Module-color theming:** Every lesson view, sidebar active item, and callout border must use the accent color of the *current module*, not a generic gold.
- **Fade on render:** Always apply `.fade-in` to the main content container when calling `renderLesson()`.
- **Video placeholders:** Keep placeholder UI visible and include HTML comments showing all three embed options (YouTube iframe, Vimeo iframe, HTML5 video). Do not remove the comments.
- **Form demo mode active by default:** The form backend integration should be commented out; demo mode (1s delay → showSuccess) should be the default state so the page works without a backend.
